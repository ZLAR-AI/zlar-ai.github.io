Get started

One command. Your agent is governed.

ZLAR installs with deny-heavy defaults. Reads and writes are allowed. Deletes, privilege escalation, network exfiltration, persistence mechanisms, and code deployment are blocked. No configuration required. No Telegram required. Risky actions are simply denied until you choose otherwise.

curl -fsSL https://zlar.ai/install.sh | bash

Works with Claude Code, Cursor, and Windsurf. Pure bash. Zero dependencies beyond openssl and jq. Governance running in under 60 seconds.

The problem

Trust is not verification.

AI agents running on your machine have access to your files, your credentials, your email, and your network. The models are powerful. The frameworks are impressive. But when it comes to governance — to the question of how you know an agent is behaving within its boundaries — the answer is: you trust it.

In financial services, healthcare, nuclear energy, and aviation, we require independent external audit because the entity performing the work cannot credibly audit itself. The same principle applies here.

ZLAR is the independent external layer. It sits between the agent and the operating system — the execution boundary where reasoning becomes action. It enforces mechanically. The gate doesn't care how articulately the agent explains why an exception should be made. Intelligence persuading intelligence is an arms race. A gate that checks paperwork is not.

Architecture

Tool calls. Intercepted. Evaluated. Decided.

When your agent tries to do anything — run a command, write a file, make a network request — ZLAR intercepts the call before it executes.

intercept
Agent makes a tool call

The framework hook fires before execution. The call never reaches the OS without passing through ZLAR.

evaluate
Gate evaluates against signed policy

Ed25519-signed rules. The agent cannot modify the rules that govern it. Tampering is cryptographically detectable.

decide
Allow, deny, or ask a human

Safe actions pass through. Dangerous actions are blocked. Ambiguous actions are routed to your phone via Telegram.

record
Every decision is audited

Append-only JSONL audit trail. Hash-chained. Neither the agent nor the operator can silently rewrite history.

Fail closed. Gate down = all actions denied.
No learning, no inference, no language understanding in the gate. It classifies and enforces. It does not decide what is safe.
Policy is a human artifact. Agents cannot modify the rules that govern them.

What's included

One product. Five capabilities.

Gate

Universal policy engine. Pure bash, zero dependencies. Intercepts tool calls across Claude Code, Cursor, and Windsurf.

Policy

Ed25519-signed rule sets. Create, sign, validate, and inspect. The human owns the rules. The machine enforces them.

Observation

Audit analysis, compliance reporting, health checks, anomaly detection, trust scoring. Read-only observation of gate behavior.

Network

Destination-level egress filtering. Blocks exfiltration targets, metadata SSRF, cloud link-local ranges. No root required.

OS Containment

Six-layer defense-in-depth for autonomous agents. User isolation, kernel sandbox, packet filter, gate daemon, signed policy, append-only audit.

Why now

Automation is automating automation.

AI agents are here. They write code, execute commands, access networks, manage files. Their capabilities are increasing faster than our ability to verify their behavior.

The instinct is to grip the wheel harder — more restrictions, more filters, more control. But the wheel is already turning on its own. The answer is not to grip harder. The answer is governance.

Governance means human authority over autonomous systems. It means signed policies that agents cannot modify. It means audit trails that record what actually happened, not what the agent says happened. It means the gate fails closed — not because we don't trust the agent, but because trust without verification is not trust. It's hope.

ZLAR exists so that agents can operate with increasing autonomy because the governance layer makes that autonomy verifiable. The governed path is the fast path. The contained agent earns more freedom, not less.
