ZLAR is a boundary between artificial cognition and worldly consequence.

An AI agent can write, call tools, move data, and trigger workflows before anyone can see who authorized it. ZLAR makes routed actions cross a separate authority — one humans control — before they become consequence.

Human problem

Action is arriving faster than accountability.

A model can recommend. An agent can do. The hard moment is not the sentence it writes, but the action it causes: a file changed, a workflow triggered, a record moved, a system called.

Governance becomes visible when a proposed action is allowed, denied, or sent to a human before it takes effect.

Receipt

If an AI agent affected you, ask for the receipt.

A receipt is the record of one governed action: what was attempted, which policy applied, what decision was made, who or what authorized it, and whether the evidence still verifies.

Disclosure

The claim is deliberately bounded.

ZLAR governs routed/intercepted action surfaces only. The public sample shows one bounded v3.3.15 proof path and names what it does not cover.

Short boundary

  • ZLAR governs routed/intercepted action surfaces only.
  • Safe Codex wording: "ZLAR can govern Codex CLI-invoked MCP tool calls when those MCP servers are routed through ZLAR."
  • Unrouted shell/filesystem/browser/app/network/model-reasoning/final-text surfaces are not claimed as governed by this proof path.
  • /contest is not implemented.
  • External non-Vincent verifier attestation remains prepared/pending unless state changes.
