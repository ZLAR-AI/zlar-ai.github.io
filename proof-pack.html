Run it

Fetch the scratch pack and run the verifier.

The commands below fetch only public fake/scratch artifacts from this website. They do not need ZLAR credentials, private keys, live Telegram, real chat IDs, raw MCP args, prompt text, final client text, or operator paths. 

mkdir zlar-proof-pack-v0-scratch
cd zlar-proof-pack-v0-scratch
mkdir -p evidence

curl -fsSLO https://zlar.ai/demo/proof-pack/README.md
curl -fsSLO https://zlar.ai/demo/proof-pack/proof-pack-manifest.json
curl -fsSLO https://zlar.ai/demo/proof-pack/SHA256SUMS
curl -fsSLO https://zlar.ai/demo/proof-pack/verify-proof-pack.mjs
curl -fsSLo evidence/governed-profile-coverage-report.json \
  https://zlar.ai/demo/proof-pack/evidence/governed-profile-coverage-report.json
curl -fsSLo evidence/governed-profile-coverage-report.txt \
  https://zlar.ai/demo/proof-pack/evidence/governed-profile-coverage-report.txt

shasum -a 256 -c SHA256SUMS
node verify-proof-pack.mjs

Expected result

A clean run ends in PASS.

The verifier checks file integrity, claim ceiling, report type, disclosure text, privacy flags, and unsafe text patterns. It does not pretend to be a production attestation.

What was verified

What the verifier checks.

Integrity

The files still match.

The SHA-256 sidecar checks the downloaded README, manifest, coverage JSON, coverage text, and verifier script.

Claim ceiling

The Codex claim stays bounded.

The manifest and report must use the exact safe wording for routed MCP tool calls.

Privacy

No private run details are included.

The scratch pack declares false for raw args, environment values, prompt text, final text, private paths, identifiers, credentials, and operator config values.

Disclosure

  • This public pack uses fake/scratch artifacts only.
  • It verifies the shape of the proof path, not a production deployment.
  • It does not provide external non-Vincent verifier attestation.
  • It does not implement /contest.

v3.3.15 claim boundary

Release state: ZLAR v3.3.15 on GitHub.

  • ZLAR governs routed/intercepted action surfaces only.
  • Safe Codex wording: "ZLAR can govern Codex CLI-invoked MCP tool calls when those MCP servers are routed through ZLAR."
  • Unrouted shell/filesystem/browser/app/network/model-reasoning/final-text surfaces are not claimed as governed by this proof path.
  • /contest is not implemented.
  • External non-Vincent verifier attestation remains prepared/pending unless state changes.
