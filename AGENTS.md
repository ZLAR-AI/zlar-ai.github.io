# ZLAR Website Agent Instructions

## Root-first context

If working inside Vincent's local ZLAR workspace and the parent files exist,
read the workspace root files before making changes:

1. `../AGENTS.md`
2. `../CODEX-OPERATING-PROTOCOL.md`
3. `../ZLAR-NORTH-STAR.md`
4. `../WORKSTREAMS.md`

If those parent files do not exist, treat this as a standalone public clone and
rely on the local repo files instead. Do not assume the local workspace North
Star exists in every clone.

`ZLAR_Website/` is the public website repo for zlar.ai / GitHub Pages. Work
here for public pages, website copy, navigation, static assets, and public
presentation.

Do not work here for core gate behavior, policy, receipts, installer/runtime
implementation, Meristem proof-chain work, or private draft exploration.

## Website Claim Boundary

Website copy must follow the current ZLAR evidence and, when available in Vincent's local workspace, the North Star.

Safe current shape:

> ZLAR governs boarding for defined routed action surfaces.

Do not imply broad AI governance, complete Codex coverage, production-installed
governance, sovereign recognition, external attestation, or unrouted-surface
coverage unless the repo evidence and public claim boundary have changed.

Use the airport model when it improves clarity: action as passenger, ZLAR as
checkpoint, receipt as boarding credential, downstream refusal as the aircraft
door refusing unrecognized boarding.

## Promotion Rule

Draft material from `../ZLAR-Draft/` is not website copy until intentionally
promoted. Before promoting draft copy:

- verify the claim against `../ZLAR_Repo/`;
- check the local claim boundary, and check the North Star claim ladder when available in Vincent's local workspace;
- preserve the public provenance rule;
- keep technical proof claims narrower than the actual evidence.
